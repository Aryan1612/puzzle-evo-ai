Evolutionary Program Synthesis for Educational Puzzles

Towards rigorous reasoning and educational impact

Combining evolutionary program synthesis with language-model reasoning to build verified, multimodal puzzle-solving assistants and a lightweight student-facing platform for Grades 1–8.

Download Proposal (PDF) Contact / Join

Primary objective

Publish a rigorous method combining evolutionary program synthesis with LLM reasoning and release dataset & code.

Timeline: 12 months (projected)

Secondary objective

Lightweight prototype for student use, logging attempts and pedagogically-structured hints.

Motivation

Puzzle-based learning improves spatial and logical reasoning. Yet adoption in classrooms is limited by lack of localized content, adaptive feedback and teacher tooling. Our project builds an automated puzzle assistant that generates verified strategies, provides structured hints, and supports multilingual deployment.

Approach (brief)

  • Represent puzzles using compact DSL/PDDL; annotate multiple valid solution paths and hint steps.
  • LLMs propose high-level solutions; evolutionary program synthesis refines candidates; symbolic interpreters verify them.
  • Self-improvement loop using student traces to refine generators and search operators.

Technical pipeline

Dataset & Benchmarks →
Representation (PDDL/DSL) →
LLM proposals →
Evolutionary synthesis + verification →
Student GUI & logging

Evaluation metrics

  • Solution correctness (executable/verified)
  • Multiplicity of solutions found
  • Human-rated hint usefulness
  • Generalization to unseen puzzle types

Planned pilot studies

Small classroom pilots (local languages: Hindi/Bengali) to measure engagement and learning impact; collect traces for iterative improvement.

Timeline (12 months)

  1. Months 1–3: Dataset curation and baseline reproduction.
  2. Months 4–6: Implement LLM + evolutionary + symbolic pipeline; initial experiments.
  3. Months 7–9: Stochastic/image-based puzzles; hint generation; human evaluation setup.
  4. Months 10–12: Pilot GUI, finalize experiments, paper write-up, public release.

Prototype / Platform (placeholder)

A lightweight, static-first student GUI (HTML/CSS/JS) will offer puzzle interaction, hint requests, and attempt logging. The demo will be linked here when ready. For now, this site acts as the canonical project landing page and documentation hub.

Prototype features (planned)

  • Adaptive difficulty & multi-strategy hints
  • Local-language support
  • Teacher dashboard (summary traces)
  • Small-scale pilot data collection
